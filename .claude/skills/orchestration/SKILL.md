# Agent Orchestration

## When to Delegate

**Delegate to specialists when:**
- Task requires deep domain expertise (engineering, design, marketing, etc.)
- Task has discrete phases (research → implement → review → test)
- Multiple independent subtasks can run in parallel
- Task is outside your core competency

**Handle yourself when:**
- Task is trivial (< 3 steps, purely conversational)
- Task is informational or requires real-time conversation

## Delegation Patterns

### Sequential (dependency chain)
```
architect → developer → reviewer → tester
researcher → strategist → writer → reviewer
```

### Parallel (independent tasks)
```
frontend-developer + backend-architect (simultaneously)
ux-researcher + trend-researcher (simultaneously)
content-creator + seo-specialist (simultaneously)
```

### Research → Build → Verify
```
codebase-explorer → specialist → reality-checker
github-code-searcher → developer → code-reviewer
```
