# Kontekst projektu

Tu trafiają notatki sesji, research, wymagania i specyfikacje.

## Przykładowe pliki

- `requirements.md` — wymagania projektu
- `session-YYYY-MM-DD.md` — notatki z sesji roboczej
- `research-[temat].md` — wyniki badań i analiz
- `specs-[feature].md` — specyfikacje funkcjonalności
