# Project Context

This folder stores session notes, research findings, requirements, and feature specifications. Agents can read and write files here to maintain context across sessions.

## Starter Templates

- [`requirements.md`](requirements.md) — project requirements and technical constraints
- [`session-template.md`](session-template.md) — copy and rename for each working session

## Suggested File Naming

- `session-YYYY-MM-DD.md` — daily working session notes
- `research-[topic].md` — research findings and analysis
- `specs-[feature].md` — feature specifications

## For Agents

Reference files in this folder for project context. When starting a new session, create a `session-YYYY-MM-DD.md` file to track goals, decisions, and blockers. Update `requirements.md` as the project evolves.
