# Architecture Decision Records (ADR)

Document architecture decisions here so both humans and agents understand the "why" behind technical choices. Each ADR captures one decision with its context, options, and consequences.

## Starter Template

- [`ADR-000-template.md`](ADR-000-template.md) — copy and rename to `ADR-NNN-short-title.md`

## For Agents

Before proposing significant architecture changes, check existing ADRs here. When making a non-obvious technical decision (choice of library, pattern, database schema), create an ADR to document the reasoning. Increment the number sequentially.

## Status Values

- `DRAFT` — decision under discussion
- `ACTIVE` — currently in effect
- `SUPERSEDED` — replaced by a newer ADR (reference the new one)
- `DEPRECATED` — no longer relevant
